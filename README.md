## Setup

This scaffold is **Next.js 15 (App Router) + TypeScript + Tailwind 3 + Recharts + Lucide**. You're free to swap things out if you have a strong reason — we evaluate the same way. The scaffold is a no-opinion baseline so you can start coding immediately.

```bash
git clone <your-fork-url>
cd verseodin-trial-2026
nvm use            # uses Node 22 per .nvmrc
npm install
npm run seed       # generates public/visits.json + public/monitoring-events.json
npm run dev        # http://localhost:3000
```

You should land on a placeholder index page with links to `/traffic` and `/actions`. Both routes are empty placeholders today — your job is to replace them with the two features.

### What the seed scripts produce

| File | Rows | Used by | Generated by |
|---|---|---|---|
| `public/visits.json` | ~100,000 `AiVisit` records over 90 days | Feature 1 (AI Traffic) | `npm run seed:visits` |
| `public/monitoring-events.json` | ~200 `MonitoringEvent` records over 60 days | Feature 2 (Action Centre) | `npm run seed:monitoring` |

Both are deterministic (seeded RNG). Re-running produces the same dataset. The fixed reference date used by both is **`2026-06-03`** — do not switch to `new Date()` if you edit the seeds.

The seeded JSON files are **`.gitignore`'d**. Each reviewer regenerates by running `npm run seed` after cloning.

### What you do NOT need to do

- No backend, no API server, no database.
- No auth, no login.
- No real third-party data — everything is mocked in the seed scripts.
- No design system / theming work. Tailwind defaults are fine.

## What you're building

Two routes in the existing scaffold:

### `/traffic` — AI Traffic Dashboard (~3h)

A single page showing which AI crawlers visited the site over 90 days, on which pages, at what volume. Stacked bar chart + Top Pages + Top Crawlers. Full spec: [`docs/FEATURE_1.md`](docs/FEATURE_1.md).

**Difficulty:** the seed produces **~100,000 visits**. Your chart must re-render in **<150ms after a legend toggle** on a mid-range laptop, and initial aggregation must complete in **<500ms**. Naive `Array.filter` inside `useMemo` keyed on legend state will not hit this. Read the eval signal at the bottom of `docs/FEATURE_1.md` carefully.

### `/actions` — Action Centre (~3h)

A triage queue of recommended actions with Accept / Dismiss + filters + Active/Dismissed tabs. State persists across reload. Full spec: [`docs/FEATURE_2.md`](docs/FEATURE_2.md).

**Difficulty:** the seed produces **~200 raw `MonitoringEvent`s of 4 event types** — *not* pre-baked `Action`s. **You write the derivation function** `deriveActions(events: MonitoringEvent[]): Action[]` that turns the raw events into 15–30 Action cards with sensible `type`, `severity`, and human-friendly copy. The UI renders the function's output. The eval signal lives in the cleanliness and extensibility of that derivation.
